This quickstart guide walks you through installing CockroachDB, starting a local cluster, and running your first SQL commands. You'll have a working CockroachDB cluster in just a few minutes.

## Before you begin

<Info>
For production deployments, consider using [CockroachDB Cloud](https://cockroachlabs.cloud/signup) for a fully managed experience.
</Info>

This guide assumes you have basic familiarity with SQL databases and command-line interfaces.

## Step 1: Install CockroachDB

Choose your operating system and follow the installation instructions:

<Tabs>
  <Tab title="macOS">
    Install CockroachDB using Homebrew:

    ```bash
    brew install cockroachdb/tap/cockroach
    ```

    Verify the installation:

    ```bash
    cockroach version
    ```

    For other installation methods, see [Install CockroachDB on Mac](/install-cockroachdb#mac).
  </Tab>
  
  <Tab title="Linux">
    Download and extract the CockroachDB archive:

    ```bash
    # Download the binary (replace {VERSION} and {ARCHITECTURE})
    curl https://binaries.cockroachdb.com/cockroach-{VERSION}.linux-{ARCHITECTURE}.tgz | tar -xz
    
    # Copy the binary to your PATH
    sudo cp -i cockroach-{VERSION}.linux-{ARCHITECTURE}/cockroach /usr/local/bin/
    ```

    Verify the installation:

    ```bash
    cockroach version
    ```

    For detailed instructions, see [Install CockroachDB on Linux](/install-cockroachdb#linux).
  </Tab>
  
  <Tab title="Windows">
    Download the executable from the [Releases page](https://www.cockroachlabs.com/docs/releases/) and extract it to a directory in your PATH.

    Verify the installation in PowerShell:

    ```powershell
    cockroach version
    ```

    For detailed instructions, see [Install CockroachDB on Windows](/install-cockroachdb#windows).
  </Tab>
  
  <Tab title="Docker">
    Pull the CockroachDB Docker image:

    ```bash
    docker pull cockroachdb/cockroach:latest-v25.3
    ```

    Verify the installation:

    ```bash
    docker run --rm cockroachdb/cockroach:latest-v25.3 version
    ```
  </Tab>
</Tabs>

## Step 2: Start a single-node cluster

For quick testing and development, start a single-node cluster:

```bash
cockroach start-single-node --insecure --listen-addr=localhost
```

<Warning>
The `--insecure` flag disables all security features. Only use this for local testing, never in production.
</Warning>

You'll see output indicating the cluster has started:

```
*
* WARNING: RUNNING IN INSECURE MODE!
*
* - Your cluster is open for any client that can access localhost.
* - Any user, even root, can log in without providing a password.
* - Any user, connecting as root, can read or write any data in your cluster.
*
CockroachDB node starting at 2026-03-03
```

## Step 3: Use the SQL shell

Open a new terminal and connect to your cluster using the built-in SQL client:

```bash
cockroach sql --insecure --host=localhost:26257
```

You'll see the SQL prompt:

```
root@localhost:26257/defaultdb>
```

## Step 4: Run your first SQL commands

Let's create a database, a table, and insert some data:

<Steps>
  <Step title="Create a database">
    ```sql
    CREATE DATABASE bank;
    ```
  </Step>
  
  <Step title="Use the database">
    ```sql
    USE bank;
    ```
  </Step>
  
  <Step title="Create a table">
    ```sql
    CREATE TABLE accounts (
      id INT PRIMARY KEY,
      balance DECIMAL
    );
    ```
  </Step>
  
  <Step title="Insert data">
    ```sql
    INSERT INTO accounts VALUES (1, 1000.50);
    INSERT INTO accounts VALUES (2, 2500.75);
    ```
  </Step>
  
  <Step title="Query the data">
    ```sql
    SELECT * FROM accounts;
    ```

    You should see:

    ```
      id | balance
    -----+---------
       1 | 1000.50
       2 | 2500.75
    (2 rows)
    ```
  </Step>
</Steps>

## Step 5: Access the DB Console

CockroachDB includes a web-based DB Console for monitoring your cluster:

1. Open your browser and navigate to [http://localhost:8080](http://localhost:8080)
2. Explore the cluster overview, metrics, and database information

<Tip>
The DB Console provides real-time insights into your cluster's health, performance, and resource usage.
</Tip>

## Step 6: Stop the cluster

When you're done, stop the cluster by pressing `Ctrl+C` in the terminal where CockroachDB is running.

To remove the data directory:

```bash
rm -rf cockroach-data/
```
